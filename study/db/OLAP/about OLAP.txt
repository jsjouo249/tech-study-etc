OLAP : DW를 활용하여 데이터 분석을 통해, 사용자에게 유의미한 정보를 제공해주는 처리 방법.

CUBE : 릴레이션 기반의 소스에서 다양한 정보에 대한 선계산 값을 미리 산출하여 저장하는 연산.
큐브에 저장하는 방법에 따라 종류가 나뉨.

ROLAP - Relational OLAP
표준 관계형 연산을 실행하기 위해 확장하여 다차원 데이터 맵핑과 함께 확장한 RDBMS
릴레이션 DB에 존재하는 데이터와 함께 동작하며, fact table과 dimension table을 생성하여 활용한다.
fact 	  : 집계의 기반이 되는 데이터
dimension : fact 테이블의 데이터 분류를 하기 위한 속성값 

장점 : 높은 데이터 효율, 확장성
단점 : 높은 자원 요구, 전체적인 데이터의 한계, 느린 쿼리 속도

MOLAP - 다차원 데이터베이스에 기반한 OLAP으로 다차원 데이터의 저장과 프로세싱.
다차원배열에 모든 조합의 데이터들을 적재하며, 해당 배열에 직접적으로 접근 가능.
네트워크 상의 데이터 이동이 최소화 되어있으며, 데이터의 저장과 프로세싱에 동일한 엔진이 사용된다.

HOLAP - Hybrid OLAP으로 ROLAP 과 MOLAP을 합친 것으로 MOLAP의 빠른 계산 + ROLAP의 높은 확장성을 동시에 제공
DB
1. 집계가 되거나 계산된 데이터들인 다차원 OLAP 큐브를 저장할 DB
2. 자세한 정보를 저장할 RDB
2개의 DB를 사용하며,

장점 : 디스크 공간 절약, 모든 타입의 데이터에 빠른 성능을 보여주는 CUBE 기술 사용, 업데이트 되는 데이터에 대해 실시간으로 접근 가능하며, 데이터 관련성을 향상시킴.
단점 : 복잡한 기능, 기능들 간의 중복 가능

OLAP을 사용하는 이유
1. 계획, 예산 정책 등 보고 및 분석을 포함한 BI 지원
2. 정보와 데이터들을 OLAP 큐브에서 일관되게 관리할 수 있음.
3. 새로운 시나리오에 대해 빠른 생성 및 분석 가능

단점
1. OLAP을 위한 눈꽃 스키마, 스타 스키마는 구현과 관리가 복잡.
2. 1개의 CUBE에 대해 많은 수의 차원들을 가질 수 없음.
3. 트랜잭션 데이터는 OLAP 시스템으로 접근이 어려움.
4. CUBE를 수정 시, 업데이트에 대한 시간과 비용이 많음.
