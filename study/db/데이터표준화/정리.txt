데이터 표준화 - 데이터 정보 요소에 대해 명칭, 정의, 형식, 규칙들의 표준을 정해 적용하는 작업
ㄴ 공공기관의 데이터베이스 표준화 지침

표준화 요소
고유성 : 데이터의 명칭은 다른 개념을 표현하는 명칭과 동일하지 않은 값
규칙성 : 명칭을 구성하는 용어, 단어들은 통일된 규칙을 가져야 함
보편성 : 명칭은 데이터를 취급하는 관점에서 보편적으로 인지되어야 하며, 업무에서 사용되는 표현을 사용
충분성 : 데이터 개념을 충분히 표현 가능 하도록 구체적

표준화
Data 항목 표준 -> 업무처리 기준 Data -> 업무처리 방법 및 절차 -> 전사 업무운영 체계


표준화 대상 : attribute, column, data-type


내가 듣고 한 것
표준용어 정의서	- 컬럼명이 되며, DB의 모든 컬럼 명은 기입되어야 하며, 도메인 정의서를 보며 한글명, 영문명, 데이터 타입, 길이를 전체적으로 통일시켜야 한다.
도메인 정의서	- 표준단어 정의서의 도메인을 사용하여 컬럼의 마지막을 도메인으로 끝마친다. 도메인 정의서에는 도메인 명, 영문 명, 설명, 데이터 타입, 길이, 도메인 분류, 출처가 필요하다.
표준단어 정의서 	- 행정표준용어가 정의된 용어들의 집합으로, 표준용어명, 영문명, 영문약어명, 도메인, 설명등이 있다.

새로운 컬럼 생성 시,
표준단어 정의서에서 표준화 할 단어들의 약어를 채택하며, 컬럼의 끝맺음 약어를 표준단어 정의서에서 찾은 뒤, 해당 끝맺음 약어를 도메인으로 설정
도메인 정의서에 표준화할 데이터 명칭의 끝맺음 말을 도메인으로 설정한 뒤, 해당 도메인에 대한 데이터 타입, 데이터 길이, 도메인 분류를 정의
표준용어 정의서에 해당 데이터 명칭의 한글, 컬럼 명, 도메인 한글명, 도메인 영문명, 데이터 타입, 길이 등 기입

찾은 결과
1. 해당 프로젝트의 상황에 맞는 현황 조사.
2. 표준화 관련 자료 수집.
3. 영문컬럼명 한글화
4. 표준화 대상 확정

데이터 사전 생성
용어 사전		- 영문의 컬럼명을 테이블 별로 한글화 하여 작성
도메인 사전	- 영문 컬럼명의 끝자리 영문 약어, 한글 컬럼명, 데이터 유형, 데이터 길이등의 정보를 활용하여 생성
단어 사전 	- 용어 사전의 영문 컬럼명과 한글 컬럼명을 토대로 단어 분리. 단어만 추출 시, 의미파악이 어렵기 때문에, 해당 테이블과 컬럼명을 출처처럼 적으면 도움
